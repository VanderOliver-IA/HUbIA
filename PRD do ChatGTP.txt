
# PRD — Plataforma SaaS de Inteligência Artificial

## Nome provisório do produto

**HUb de IA OMD*

Sistema SaaS de Inteligência Artificial multi-tenant para agências, empresas e criadores.

---

# 1️⃣ Visão do Produto

Criar uma **plataforma SaaS de Inteligência Artificial multi-tenant** que permita:

* centralizar múltiplas IAs
* treinar contexto por empresa
* criar agentes especializados
* automatizar fluxos
* permitir uso independente por clientes

A plataforma será operada pela **OMD como super administradora**.

---

# 2️⃣ Estrutura de Usuários do Sistema

## 1️⃣ Super Admin (OMD)

Controle total da plataforma.

Pode:

* criar contas
* criar clientes
* inserir contexto
* criar agentes
* gerenciar APIs
* acessar analytics

---

## 2️⃣ Cliente da Agência

Exemplo:

* Oficina Belas Artes
* Sindicos Awards
* outros clientes

Esses clientes usam o sistema **já treinado com seus dados**.

---

## 3️⃣ Usuário SaaS Independente

Usuário que compra acesso ao sistema mas **não é cliente da OMD**.

Características:

* ambiente isolado
* dados privados
* OMD não acessa conteúdos internos

---

# 3️⃣ Conceito de Tenant (fundamental)

Cada empresa será um **tenant isolado**.

Exemplo:

```
Sistema
   │
   ├─ OMD (super admin)
   │
   ├─ Oficina Belas Artes
   │
   ├─ Sindicos Awards
   │
   └─ Cliente SaaS Independente
```

Todos usam o mesmo software, mas:

* dados
* memória
* documentos
* branding

são **separados**.

Essa arquitetura é chamada de **multi-tenant SaaS**. ([NTCONSULT][1])

## Funcionamento

Quando um usuário seleciona um cliente:

```
Selecionar cliente → Oficina Belas Artes
```

O sistema automaticamente carrega:

```
contexto
documentos
estratégia
branding
tom de voz
```

E todas as IAs passam a responder **baseadas nesse contexto**.

# 7️⃣ Painel do Usuário SaaS Independente

Usuário que compra o sistema.


E a OMD **não terá acesso aos dados dele**.
